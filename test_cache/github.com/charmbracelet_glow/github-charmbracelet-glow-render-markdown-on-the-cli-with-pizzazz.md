Glow is a terminal based markdown reader designed from the ground up to bring out the beauty‚Äîand power‚Äîof the CLI.

Use it to discover markdown files, read documentation directly on the command line. Glow will find local markdown files in subdirectories or a local Git repository.

Installation
------------

[](https://github.com/charmbracelet/glow?screenshot=true#installation)

### Package Manager

[](https://github.com/charmbracelet/glow?screenshot=true#package-manager)

```shell
# macOS or Linux
brew install glow

# macOS (with MacPorts)
sudo port install glow

# Arch Linux (btw)
pacman -S glow

# Void Linux
xbps-install -S glow

# Nix shell
nix-shell -p glow --command glow

# FreeBSD
pkg install glow

# Solus
eopkg install glow

# Windows (with Chocolatey, Scoop, or Winget)
choco install glow
scoop install glow
winget install charmbracelet.glow

# Android (with termux)
pkg install glow

# Debian/Ubuntu
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://repo.charm.sh/apt/gpg.key | sudo gpg --dearmor -o /etc/apt/keyrings/charm.gpg
echo "deb [signed-by=/etc/apt/keyrings/charm.gpg] https://repo.charm.sh/apt/ * *" | sudo tee /etc/apt/sources.list.d/charm.list
sudo apt update && sudo apt install glow

# Fedora/RHEL
echo '[charm]
name=Charm
baseurl=https://repo.charm.sh/yum/
enabled=1
gpgcheck=1
gpgkey=https://repo.charm.sh/yum/gpg.key' | sudo tee /etc/yum.repos.d/charm.repo
sudo yum install glow
```

Or download a binary from the [releases](https://github.com/charmbracelet/glow/releases) page. MacOS, Linux, Windows, FreeBSD and OpenBSD binaries are available, as well as Debian, RPM, and Alpine packages. ARM builds are also available for macOS, Linux, FreeBSD and OpenBSD.

go install github.com/charmbracelet/glow@latest

The TUI
-------

[](https://github.com/charmbracelet/glow?screenshot=true#the-tui)

Simply run `glow` without arguments to start the textual user interface and browse local. Glow will find local markdown files in the current directory and below or, if you‚Äôre in a Git repository, Glow will search the repo.

Markdown files can be read with Glow's high-performance pager. Most of the keystrokes you know from `less` are the same, but you can press `?` to list the hotkeys.

The CLI
-------

[](https://github.com/charmbracelet/glow?screenshot=true#the-cli)

In addition to a TUI, Glow has a CLI for working with Markdown. To format a document use a markdown source as the primary argument:

```shell
# Read from file
glow README.md

# Read from stdin
echo "[Glow](https://github.com/charmbracelet/glow)" | glow -

# Fetch README from GitHub / GitLab
glow github.com/charmbracelet/glow

# Fetch markdown from HTTP
glow https://host.tld/file.md
```

### Word Wrapping

[](https://github.com/charmbracelet/glow?screenshot=true#word-wrapping)

The `-w` flag lets you set a maximum width at which the output will be wrapped:

```shell
glow -w 60
```

### Paging

[](https://github.com/charmbracelet/glow?screenshot=true#paging)

CLI output can be displayed in your preferred pager with the `-p` flag. This defaults to the ANSI-aware `less -r` if `$PAGER` is not explicitly set.

### Styles

[](https://github.com/charmbracelet/glow?screenshot=true#styles)

You can choose a style with the `-s` flag. When no flag is provided `glow` tries to detect your terminal's current background color and automatically picks either the `dark` or the `light` style for you.

```shell
glow -s [dark|light]
```

Alternatively you can also supply a custom JSON stylesheet:

```shell
glow -s mystyle.json
```

For additional usage details see:

```shell
glow --help
```

Check out the [Glamour Style Section](https://github.com/charmbracelet/glamour/blob/master/styles/gallery/README.md) to find more styles. Or [make your own](https://github.com/charmbracelet/glamour/tree/master/styles)!

The Config File
---------------

[](https://github.com/charmbracelet/glow?screenshot=true#the-config-file)

If you find yourself supplying the same flags to `glow` all the time, it's probably a good idea to create a config file. Run `glow config`, which will open it in your favorite $EDITOR. Alternatively you can manually put a file named `glow.yml` in the default config path of you platform. If you're not sure where that is, please refer to `glow --help`.

Here's an example config:

```yaml
# style name or JSON path (default "auto")
style: "light"
# mouse wheel support (TUI-mode only)
mouse: true
# use pager to display markdown
pager: true
# at which column should we word wrap?
width: 80
# show all files, including hidden and ignored.
all: true
```

is it?\n-----------\n\n[](https://github.com/charmbracelet/glow?screenshot=true#what-is-it)\n\nGlow is a terminal based markdown reader designed from the ground up to bring out the beauty‚Äîand power‚Äîof the CLI.\n\nUse it to discover markdown files, read documentation directly on the command line. Glow will find local markdown files in subdirectories or a local Git repository.\n\nInstallation\n------------\n\n[](https://github.com/charmbracelet/glow?screenshot=true#installation)\n\n### Package Manager\n\n[](https://github.com/charmbracelet/glow?screenshot=true#package-manager)\n\n```shell\n# macOS or Linux\nbrew install glow\n\n# macOS (with MacPorts)\nsudo port install glow\n\n# Arch Linux (btw)\npacman -S glow\n\n# Void Linux\nxbps-install -S glow\n\n# Nix shell\nnix-shell -p glow --command glow\n\n# FreeBSD\npkg install glow\n\n# Solus\neopkg install glow\n\n# Windows (with Chocolatey, Scoop, or Winget)\nchoco install glow\nscoop install glow\nwinget install charmbracelet.glow\n\n# Android (with termux)\npkg install glow\n\n# Debian/Ubuntu\nsudo mkdir -p /etc/apt/keyrings\ncurl -fsSL https://repo.charm.sh/apt/gpg.key | sudo gpg --dearmor -o /etc/apt/keyrings/charm.gpg\necho \"deb [signed-by=/etc/apt/keyrings/charm.gpg] https://repo.charm.sh/apt/ * *\" | sudo tee /etc/apt/sources.list.d/charm.list\nsudo apt update && sudo apt install glow\n\n# Fedora/RHEL\necho '[charm]\nname=Charm\nbaseurl=https://repo.charm.sh/yum/\nenabled=1\ngpgcheck=1\ngpgkey=https://repo.charm.sh/yum/gpg.key' | sudo tee /etc/yum.repos.d/charm.repo\nsudo yum install glow\n```\n\nOr download a binary from the [releases](https://github.com/charmbracelet/glow/releases) page. MacOS, Linux, Windows, FreeBSD and OpenBSD binaries are available, as well as Debian, RPM, and Alpine packages. ARM builds are also available for macOS, Linux, FreeBSD and OpenBSD.\n\n### Go\n\n[](https://github.com/charmbracelet/glow?screenshot=true#go)\n\nOr just install it with `go`:\n\n```shell\ngo install github.com/charmbracelet/glow@latest\n```\n\n### Build (requires Go 1.21+)\n\n[](https://github.com/charmbracelet/glow?screenshot=true#build-requires-go-121)\n\n```shell\ngit clone https://github.com/charmbracelet/glow.git\ncd glow\ngo build\n```\n\nThe TUI\n-------\n\n[](https://github.com/charmbracelet/glow?screenshot=true#the-tui)\n\nSimply run `glow` without arguments to start the textual user interface and browse local. Glow will find local markdown files in the current directory and below or, if you‚Äôre in a Git repository, Glow will search the repo.\n\nMarkdown files can be read with Glow's high-performance pager. Most of the keystrokes you know from `less` are the same, but you can press `?` to list the hotkeys.\n\nThe CLI\n-------\n\n[](https://github.com/charmbracelet/glow?screenshot=true#the-cli)\n\nIn addition to a TUI, Glow has a CLI for working with Markdown. To format a document use a markdown source as the primary argument:\n\n```shell\n# Read from file\nglow README.md\n\n# Read from stdin\necho \"[Glow](https://github.com/charmbracelet/glow)\" | glow -\n\n# Fetch README from GitHub / GitLab\nglow github.com/charmbracelet/glow\n\n# Fetch markdown from HTTP\nglow https://host.tld/file.md\n```\n\n### Word Wrapping\n\n[](https://github.com/charmbracelet/glow?screenshot=true#word-wrapping)\n\nThe `-w` flag lets you set a maximum width at which the output will be wrapped:\n\n```shell\nglow -w 60\n```\n\n### Paging\n\n[](https://github.com/charmbracelet/glow?screenshot=true#paging)\n\nCLI output can be displayed in your preferred pager with the `-p` flag. This defaults to the ANSI-aware `less -r` if `$PAGER` is not explicitly set.\n\n### Styles\n\n[](https://github.com/charmbracelet/glow?screenshot=true#styles)\n\nYou can choose a style with the `-s` flag. When no flag is provided `glow` tries to detect your terminal's current background color and automatically picks either the `dark` or the `light` style for you.\n\n```shell\nglow -s [dark|light]\n```\n\nAlternatively you can also supply a custom JSON stylesheet:\n\n```shell\nglow -s mystyle.json\n```\n\nFor additional usage details see:\n\n```shell\nglow --help\n```\n\nCheck out the [Glamour Style Section](https://github.com/charmbracelet/glamour/blob/master/styles/gallery/README.md) to find more styles. Or [make your own](https://github.com/charmbracelet/glamour/tree/master/styles)!\n\nThe Config File\n---------------\n\n[](https://github.com/charmbracelet/glow?screenshot=true#the-config-file)\n\nIf you find yourself supplying the same flags to `glow` all the time, it's probably a good idea to create a config file. Run `glow config`, which will open it in your favorite $EDITOR. Alternatively you can manually put a file named `glow.yml` in the default config path of you platform. If you're not sure where that is, please refer to `glow --help`.\n\nHere's an example config:\n\n```yaml\n# style name or JSON path (default \"auto\")\nstyle: \"light\"\n# mouse wheel support (TUI-mode only)\nmouse: true\n# use pager to display markdown\npager: true\n# at which column should we word wrap?\nwidth: 80\n# show all files, including hidden and ignored.\nall: true\n```\n\nFeedback\n--------\n\n[](https://github.com/charmbracelet/glow?screenshot=true#feedback)\n\nWe‚Äôd love to hear your thoughts on this project.
